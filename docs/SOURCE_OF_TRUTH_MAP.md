# Source Of Truth Map

Last updated: 2026-05-27 PT

Issue #80 owns this map.

## Canonical Boot Sequence

For any Foldera task, use this order:

1. Read `ACTIVE_HANDOFF.md`.
2. Read `FOLDERA_LAUNCH_ROADMAP.md`.
3. Read the active issue named by `ACTIVE_HANDOFF.md`.
4. Read issue #48 for product doctrine.
5. Read relevant execution/proof docs only for the active seam.
6. Check latest open PRs and recent merged PRs when repo/deploy truth matters.
7. Use Vercel/Supabase only when the seam requires live/runtime truth.

## Enforcement

`npm run gate:continuity` fails when:

- a required source-truth file is missing
- the canonical boot sequence drifts across active agent docs
- `ACTIVE_HANDOFF.md` does not name exactly one active seam line
- issue #48 or `FOLDERA_LAUNCH_ROADMAP.md` falls out of the handoff
- stale active-looking docs lack authority headers
- `.foldera-contract.json` still points at an old issue while treated active
- the README regresses to default Next.js boilerplate
- PR Sentinel stops running the continuity gate

This map does not authorize product, backend, auth, Slack, Supabase, Stripe, landing, dashboard, or scoring changes.
